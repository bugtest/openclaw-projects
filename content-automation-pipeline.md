## 🔧 核心模块设计

### 模块 1：内容抓取器 (Crawler)

**功能：**
- 支持多种源：URL、RSS、API
- 可配置选择器（CSS/XPath）提取目标内容
- 去重机制（URL 哈希/内容指纹）
- 失败重试 + 错误日志

**技术选型：**
- Python + Playwright/BeautifulSoup
- 或 Node.js + Puppeteer
- 定时：cron / GitHub Actions / OpenClaw cron

---

### 模块 2：飞书多维表格集成

**功能：**
- 自动创建/更新记录
- 字段映射（标题、正文、链接、抓取时间、标签、状态）
- 支持附件（图片下载后上传飞书云文档）

**数据表结构建议：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | 文本 | 文章/内容标题 |
| 正文 | 长文本 | 提取的正文内容 |
| 来源 URL | 链接 | 原始链接 |
| 抓取时间 | 日期 | 自动记录 |
| 标签 | 多选 | 分类标签 |
| 处理状态 | 单选 | 待处理/已加工/已发布 |
| AI 摘要 | 长文本 | 后续加工生成 |
| 发布平台 | 多选 | 微博/小红书/掘金 |
| 发布时间 | 日期 | 各平台发布时间 |

---

### 模块 3：智能加工引擎

**功能：**
- 定时扫描"待处理"记录
- 调用 LLM 生成：
  - 摘要/要点提炼
  - 多平台适配文案（微博短文案、小红书种草风、掘金技术文）
  - 标签自动分类
- 人工审核开关（可选）
- 更新飞书记录状态

---

### 模块 4：RAG 集成

**功能：**
- 将加工后的内容向量化
- 存入向量数据库（Chroma/Milvus/Pinecone）
- 支持语义检索
- 可对接个人知识库问答

**流程：**
```
飞书记录 → 提取正文 → AI 清洗 → 向量化 → 存入向量库
```

---

### 模块 5：自媒体发布器

**功能：**
- 微博：调用微博开放 API（需申请）
- 小红书：目前无官方 API，可用浏览器自动化（Playwright）
- 掘金：有开放 API，支持 Markdown 发布

**注意：**
- 各平台发布频率限制
- 内容合规性检查
- 发布后回写飞书记录（状态 + 链接）

## 🛠️ 技术栈汇总

## ⚠️ 风险点

1. **平台 API 限制** — 微博/小红书自动化可能触发风控
2. **内容版权** — 抓取内容需注意版权声明，建议做二次创作
3. **账号安全** — 自动化发布建议用小号测试
4. **成本控制** — AI 调用次数、向量库存储成本
